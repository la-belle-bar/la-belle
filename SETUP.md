# La Belle — настройка безопасного деплоя (GitHub Pages + Google Apps Script)

## Как это устроено

```
Покупатель ──► GitHub Pages (статичный сайт, БЕЗ секретов)
                    │
                    ▼  fetch (каталог, заказы)
              Google Apps Script  ◄── все секреты здесь (Script Properties)
                    │
                    ▼  от имени владельца
              ПРИВАТНАЯ Google-таблица + Telegram-бот
```

Сайт на GitHub Pages полностью публичный (и код, и репозиторий), поэтому в нём
нет ни одного секрета. Всё, что требует прав, делает Google Apps Script — он
выполняется от имени владельца таблицы, а админ-операции защищены ключом
`ADMIN_TOKEN`, который проверяется на сервере при каждом запросе.

---

## Шаг 1. Сделайте таблицу приватной

Google Sheets → **Настройки доступа** → уберите «Все, у кого есть ссылка».
Доступ должен остаться только у вас. Публиковать таблицу в интернете
(«Файл → Открыть доступ → Публикация») тоже **не нужно** — если публиковали,
отмените публикацию. Сайту прямой доступ к таблице больше не требуется.

## Шаг 2. Обновите Google Apps Script

1. Откройте таблицу → **Расширения → Apps Script** (или существующий проект
   скрипта, если он уже был).
2. Замените всё содержимое `Code.gs` на файл [apps-script/Code.gs](apps-script/Code.gs)
   из этого репозитория.
3. **Project Settings (шестерёнка) → Script Properties** — добавьте свойства:

   | Свойство | Значение |
   |---|---|
   | `SPREADSHEET_ID` | `1Jyrtv0BnD03CBCqCbfF1puSiAVTqG4p-MPSaXCYFOgM` |
   | `CATALOG_SHEET_GID` | `1821735406` |
   | `ORDERS_SHEET_GID` | `926886605` |
   | `ADMIN_TOKEN` | длинный случайный ключ, см. ниже |
   | `TELEGRAM_BOT_TOKEN` | *(необязательно)* токен бота от @BotFather |
   | `TELEGRAM_CHAT_ID` | *(необязательно)* id чата для уведомлений |

   `SPREADSHEET_ID` — из URL таблицы (`.../spreadsheets/d/<ВОТ_ЭТО>/edit`),
   gid листа — число после `#gid=` при открытом нужном листе.

4. Сгенерируйте `ADMIN_TOKEN` (это ваш «пароль» от админки — 32+ случайных
   символов). В PowerShell:

   ```powershell
   -join ((48..57)+(65..90)+(97..122) | Get-Random -Count 40 | ForEach-Object {[char]$_})
   ```

   Никому его не показывайте и **никогда не вставляйте в код сайта**.

## Шаг 3. Задеплойте веб-приложение

1. **Deploy → New deployment → Web app** (тип: Web app).
2. **Execute as: Me** (важно — скрипт работает от вашего имени).
3. **Who has access: Anyone** (это открывает только API скрипта, не таблицу).
4. Скопируйте URL вида `https://script.google.com/macros/s/…/exec`.

Если деплой уже существовал, обновите его: **Deploy → Manage deployments →
✏️ Edit → Version: New version → Deploy** — тогда URL не изменится.

## Шаг 4. Пропишите URL в сайте

В [assets/js/config.js](assets/js/config.js) укажите ваш URL в `apiUrl`.
Это единственная настройка на стороне сайта. Секретов в этом файле быть
не должно.

## Шаг 5. Telegram-уведомления (по желанию)

1. Создайте бота у [@BotFather](https://t.me/BotFather) → получите токен.
2. Добавьте бота в нужный чат/группу (или напишите ему в личку).
3. Узнайте chat id: откройте `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
   после отправки сообщения в чат — в ответе будет `"chat":{"id":…}`.
4. Впишите `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` в Script Properties.

Уведомления отправляет сервер при приёме заказа — токен бота в браузер
не попадает.

## Шаг 6. GitHub Pages

1. Repository → **Settings → Pages** → включите **Enforce HTTPS**.
2. Помните: на бесплатном тарифе репозиторий для Pages публичный. Это
   нормально — в коде секретов нет. Но **не коммитьте** реальные токены даже
   «на минутку»: git-история публична навсегда. Если токен попал в историю —
   просто смените `ADMIN_TOKEN` в Script Properties.

## Как пользоваться админкой

Откройте `admin.html`, введите `ADMIN_TOKEN`. Ключ проверяется сервером и
хранится только в текущей вкладке (sessionStorage) до выхода/закрытия.
Всё, что делает админка (заказы, статусы, правка товаров), Apps Script
выполняет только при верном ключе.

## Что защищено и от чего

- **Данные клиентов** (телефоны, адреса) лежат в приватной таблице и отдаются
  только по админ-ключу. Публичного CSV-экспорта больше нет.
- **Правка каталога и статусов** — только с ключом, проверка на сервере.
- **Подмена цены в браузере** — сервер сверяет цены заказа с каталогом и пишет
  результат в колонку `price_check` листа заказов: заказ с подменённой ценой
  будет помечен `MISMATCH!` (и выделен в Telegram-уведомлении). Проверяйте её
  до подтверждения заказа.
- **Спам заказами** — скрытое honeypot-поле против ботов + общий лимит
  30 заказов / 10 минут на сервере.
- **XSS** — строгая Content-Security-Policy на всех страницах + экранирование
  HTML во всём рендеринге.

## Ограничения (осознанные)

- Лимит Apps Script для личного аккаунта — до ~20 000 запросов/день к веб-апп.
  Для небольшого магазина это с запасом.
- «Вход» в админку — это проверка ключа сервером; сам файл admin.html публичен
  (это просто интерфейс). Без ключа он не может ни прочитать, ни изменить
  ничего.
- Заказ, оформленный покупателем, дублируется в localStorage его браузера —
  это только его локальная история, другим она не видна.
